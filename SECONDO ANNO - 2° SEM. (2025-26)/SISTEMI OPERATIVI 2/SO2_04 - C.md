Il linguaggio **C** è probabilmente uno dei linguaggi di programmazione più importanti, se non **il linguaggio di programmazione più importante** mai creato, a livello di popolarità e di quante cose sono state create utilizzandolo.

È stato sviluppato da AT&T Bell Labs agli inizi degli anni '70, e già uno dei suoi primissimi scopi è stato la **creazione del kernel UNIX**. In seguito, è stato utilizzato anche per sviluppare **Linux**, così come altri sistemi operativi. Il C è stato, eventualmente, **standardizzato dall'ANSI** (American National Standards Institute) **e dall'ISO** (International Organization for Standardization).

## Ciclo di vita di programma in C

Similmente ad altri linguaggi come Java, **i programmi C sono indipendenti dall'hardware**, dunque possono essere eseguiti senza, o con pochissime, modifiche su ogni tipo di processore (previa eventuale ricompilazione).

Si tratta di un **linguaggio compilato**, dunque il codice sorgente di un programma C, per poter essere eseguito, dovrà prima passare per una serie di passaggi:
- **scrittura del codice**;
- **pre-processamento**;
- **compilazione**;
- **linking**.

Il **pre-processamento** viene effettuato da uno strumento chiamato **`cpp`**, ed è concretamente il primo strumento a toccare il codice dopo la scrittura. Il suo compito è quello di **rimuovere i commenti** e di **eseguire le direttive**, ossia tutte quelle istruzioni che cominciano con un hashtag (**`#`**), che consistono principalmente nell'**inclusione di header files** (si tratta di file molto importanti, che consistono in vere e proprie "librerie" di funzioni; terminano sempre con **`.h`**, e alcuni dei più importanti sono **`stdio.h`** per l'I/O, **`stdlib.h`** per varie funzionalità tra cui l'[[SO2_04 - C#Allocazione dinamica di memoria|allocazione dinamica di memoria]], o anche **`string.h`** per la gestione delle [[SO2_04 - C#Stringhe|stringhe]]).

La **compilazione** viene effettuata da uno strumento chiamato **`gcc`**, e consiste prima in un **controllo rigoroso della sintassi** del codice, così come delle **intestazioni delle chiamate a funzione**, e poi nella **traduzione del codice in "codice oggetto"**, che verrà poi salvato in un file **`.o`** e che consiste sostanzialmente in una versione in linguaggio macchina del codice originale. Tuttavia, ogni chiamata a funzione contenuta nel codice avrà a questo punto una destinazione puramente simbolica, e per completare il processo ci sarà bisogno della fase di linking.

Il **linking** viene effettuato da uno strumento chiamato **`ld`**, e consiste nell'**assemblaggio finale dell'eseguibile**, risolvendo le chiamate a funzione e collegando eventuali funzioni dipendenti su [[SO2_05 - System calls|chiamate di sistema]] con le relative librerie di sistema. Inoltre, se il progetto è diviso in più file `.o`, il linker `ld` contribuisce a creare un unico file eseguibile finale.

Una volta ottenuto l'eseguibile, siamo pronti per l'**esecuzione** del programma. Per fare ciò, sarà necessario far intervenire un ultimo strumento, controllato dall'OS: il **loader**. Supponendo di voler eseguire un file chiamato `a.out` contenuto nella cartella corrente, scriveremmo sul [[SO2_01 - Shell|terminale]]:

```
./a.out
```

dove `./` indica al terminale di cercare il programma `a.out` nella cartella corrente; al momento dell'esecuzione di tale comando, il loader si attiva per **caricare il file eseguibile dal disco nella memoria principale**. A questo punto, la CPU potrà prelevare in modo sequenziale le istruzioni da eseguire e procedere con l'esecuzione concreta del programma indicato.

Ognuno dei passaggi approfonditi poco fa può essere eseguito singolarmente su un file C tramite specifici **comandi da terminale**. Ad esempio, è possibile effettuare pre-compilazione, compilazione e linking di un file `file.c` tutti insieme semplicemente con il comando:

```
gcc file.c
```

È possibile aggiungere varie **[[SO2_01 - Shell#Approfondimento sulle opzioni|opzioni]]** a tale comando per ottenere risultati leggermente diversi: ad esempio, aggiungere **`-Wall`** prima del nome del file da compilare **impone al compilatore di stampare tutti i messaggi di errore o di avvertimento**, se ci sono; invece, nel caso in cui il programma contenga la libreria matematica **`math.h`**, si dovrà aggiungere `-lm` dopo il nome del file; ancora, se si vuole specificare il nome dell'eseguibile risultante si potrà aggiungere **`-o`**, seguito dal nome desiderato, dopo il nome del file da compilare.

Se volessimo, invece, effettuare **solo la pre-compilazione** di un file `file.c`, andremmo ad eseguire:

```
cpp file.c > file_precompilato.c
```

Volendo effettuare **solo la compilazione di un file pre-compilato** `file_precompilato.c`, si eseguirebbe:

```
gcc -c file_precompilato.c
```

Ancora, se volessimo effettuare **solo la pre-compilazione e la compilazione** di un file `file.c`, si eseguirebbe:

```
gcc -c file.c
```

Infine, volendo effettuare **solo il linking** di un file eseguibile `file.o`, si eseguirebbe:

```
gcc file.o
```

## Funzioni

Le **funzioni**, costrutto comunissimo e utilissimo in qualsiasi linguaggio di programmazione, assumono un'importanza fondamentale in C per un semplice motivo: **qualsiasi istruzione di un programma in C si troverà tendenzialmente all'interno di una funzione**. Ma cos'è, concretamente, una funzione?

In parole povere, possiamo vedere una funzione come un **blocco di codice riutilizzabile**, che esegue un compito specifico e che è identificato da un nome univoco. A livello di **sintassi**, ogni funzione consiste di un'**intestazione** (detta anche "**header**") e di un **blocco di codice**, trovati tipicamente nella seguente configurazione:

```
return_type function_name(parameters) 
{
	basic block
}
```

Nel `basic block`, ossia il blocco di codice della funzione, sarà inserito tutto il codice che definirà il **funzionamento della funzione**; per comodità e chiarezza, è consigliato **dichiarare tutte le variabili necessarie prima di scrivere il codice effettivo**, sia per un maggiore ordine sia per avere fin da subito tutti gli strumenti per definire il flusso di esecuzione della funzione.

Come si può notare dal prototipo di funzione appena fornito, ogni funzione ha un **tipo di ritorno**: infatti, generalmente, al termine del basic block di una qualsiasi funzione ci sarà un'istruzione **`return`**, tipicamente inserita secondo la forma:

```
return expression;
```

dove `expression` può essere una costante, una variabile, un'espressione logico-aritmetica o anche un'altra funzione. **L'espressione ritornata dalla funzione sarà poi fornita alla funzione chiamante** come "risultato" della chiamata (è perfettamente possibile, d'altronde, che una funzione non ritorni nulla, caso in cui sarà assente l'istruzione `return` e il tipo di ritorno sarà **`void`**).

Come accennato in precedenza, le funzioni sono fondamentali per la programmazione in C. Basti pensare alla **funzione `main()`**, obbligatoria all'interno di un programma C, che sarà il **punto di partenza dell'esecuzione del programma** e che, nel concreto, conterrà tutte le invocazioni effettive delle eventuali funzioni dichiarate nello stesso, o nel caso di programmi più semplici conterrà direttamente tutto il codice del programma. A seconda del programma, il `main()` potrà essere **privo di parametri** oppure averne **due**:
- **`int argc`**, che corrisponde al numero di "parole" eventualmente passate al programma, tramite terminale, al momento della sua esecuzione;
- **`char *argv[]`**, che corrisponde all'[[SO2_04 - C#Array|array]] delle "parole" eventualmente passate al programma, tramite terminale, al momento della sua esecuzione.

Una precisazione molto importante è che **il comando scritto per eseguire il programma viene considerato come primo elemento di `argv[]`**: dunque, nell'eventualità in cui si inseriscano $n$ argomenti, `argc` sarà uguale a $n+1$. Ad esempio, eseguendo da terminale un programma con gli argomenti:

```
ciao come stai?
```

`argc` sarà uguale a 4, mentre `argv` conterrà, nelle posizioni dalla `1` alla `3`, rispettivamente le parole `ciao`, `come` e `stai?`.

## Input e output

Nel parlare di C, e in generale di qualsiasi forma di programmazione, è fondamentale avere ben chiari i concetti di "**input**" e "**output**" di un programma. Con questi due termini, si intendono due cose ben precise:
- un programma che richiede un **input** dall'utente tipicamente andrà a **chiedere all'utente** (durante la sua esecuzione) **di inserire dei dati tramite la tastiera o qualche altro dispositivo**, dati che verranno poi memorizzati dal programma in esecuzione all'interno di determinate variabili;
- un programma che restituisce un **output** all'utente tipicamente va a **stampare a schermo, o inviare a un dispositivo, dei dati**, che possono essere "fissi" o risultati di operazioni svolte durante l'esecuzione del programma.

Essendo due concetti non solo molto importanti, ma anche strettamente legati, spesso ci si riferisce a input e output parlando di "**I/O**". In C, tutte le **funzioni essenziali per l'I/O** sono contenute nel file **`stdio.h`**.

##### Output

Il principale metodo con cui un programma **restituisce un output** è tramite la **stampa a schermo**: si specifica il dato da stampare ed esso verrà visualizzato sullo schermo, tipicamente nel terminale. Per stampare a schermo qualcosa in un programma C, si utilizza la funzione:

```
printf(format-string, value-list);
```

dove `format-string` rappresenta la stringa di testo che si vuole stampare, e in particolare si tratta di una "**stringa formattata**", il che vuol dire che è una **stringa di testo che può contenere anche dei segnaposti**, che al momento della stampa verranno poi **sostituiti da valori specifici in base alle espressioni specificate in ordine in `value-list`**. Quest'ultima può contenere pressoché qualsiasi cosa:
- sequenze di caratteri;
- [[SO2_04 - C#Variabili e operatori|variabili]];
- [[SO2_04 - C#Dichiarazione e tipi di dati|costanti]];
- espressioni [[SO2_04 - C#Operatori|logico-matematiche]].

Inoltre, è importante ricordare che **la funzione `printf` restituisce il numero di caratteri stampati**.

Un **segnaposto** inizia sempre con un simbolo percentuale (**`%`**), e in base al carattere o alla sequenza di caratteri che segue tale simbolo si indicherà al compilatore il **tipo di dato che verrà stampato al posto del segnaposto**. In particolare:
- **`%d`**, o anche **`%i`**, viene utilizzato per gli **interi di tipo `int`**;
- **`%l`** viene utilizzato per gli **interi di tipo `long`**;
- **`%x`** viene utilizzato per gli **interi espressi in esadecimale**;
- **`%o`** viene utilizzato per gli **interi espressi in ottale**;
- **`%f`** (formato standard), **`%e`** (notazione scientifica) e **`%g`** (sceglie in automatico il formato migliore tra `%f` e `%e`) vengono utilizzati per i **numeri a virgola mobile**;
- **`%lf`** viene utilizzato per i **numeri a virgola mobile di tipo `double`**;
- **`%s`** viene utilizzato per le **[[SO2_04 - C#Stringhe|stringhe]]**;
- **`%c`** viene utilizzato per i **caratteri** singoli.

Vediamo un esempio per capire meglio il funzionamento dei segnaposti nella funzione `printf`. Eseguendo il seguente codice:

```
int num1 = 55;
int num2 = 30;
int sum = num1 + num2;

int printCount;

printCount = printf("%d + %d = %d\n", num1, num2, sum);
```

avremo nella variabile `sum` il numero `85`, e dunque la funzione `printf` stamperà a schermo la seguente stringa di testo:

```
55 + 30 = 85
```

mentre la variabile `printCount` assumerà il valore `13`, dato che sono 13 i caratteri che andranno a comporre la stringa finale da stampare.

In realtà, i segnaposti possono diventare molto più complessi di quelli visti finora. Basti pensare che il **formato completo di un segnaposto** generico è in realtà il seguente:

```
%[parameter][flags][width][.precision][length]type
```

La parte **`type`** coincide con il **tipo di dato**, ossia i caratteri o sequenze di caratteri viste poco fa. Vediamo, a questo punto, in cosa consiste ciascuna delle opzioni facoltative. 

L'opzione **`parameter`** permette di **selezionare esplicitamente un argomento specifico tra quelli specificati nella `value-list`, ignorando l'ordine sequenziale**. Per fare ciò, si indica la posizione `n` dell'argomento da considerare, seguito dal simbolo del dollaro (**`$`**): ad esempio, il segnaposto `%2$d` formatterà il secondo argomento presente nella `value-list`, considerandolo come intero.

L'opzione **`flags`** consiste in uno o più simboli tra i seguenti (inseribili in qualsiasi ordine):
- **`-`**, che impone di **allineare l'output a sinistra dello spazio prestabilito** (di default, l'allineamento avviene a destra);
- **`+`**, che impone di **anteporre il simbolo `+` a eventuali numeri positivi**, dato che di default i numeri positivi non presentano segno visibile;
- **` `**, che impone di **anteporre uno spazio vuoto a eventuali numeri positivi** (viene ignorato se è presente anche il simbolo `+` tra le `flags`);
- **`0`**, che se utilizzato in combinazione con l'opzione `width` permette di **riempire lo spazio vuoto iniziale con degli zeri** invece che con degli spazi (si capirà meglio di cosa si sta parlando quando si parlerà dell'opzione `width` più nel dettaglio);
- **`'`**, che impone di **aggiungere un separatore ogni tre cifre in numeri interi o a virgola mobile**;
- **`#`**, che permette di **attivare un formato alternativo per l'argomento** in base al suo tipo di dato (ad esempio, per argomenti di tipo `o`, `x` e `X` aggiunge in automatico rispettivamente i prefissi `0`, `0x` e `0X`, per argomenti di tipo `f` ed `e` costringe al programma di visualizzare il punto decimale, mentre per argomenti di tipo `g` e `G` impedisce la rimozione di eventuali zeri finali non significativi).

L'opzione **`width`** permette di **indicare il numero minimo di caratteri che l'argomento formattato dal segnaposto dovrà occupare**. Dunque, nel caso in cui l'argomento da stampare occupa meno caratteri del valore `width` indicato, lo spazio restante verrà creato da spazi vuoti, o da zeri se inserito il simbolo `0` tra le `flags`, posti alla sinistra dello stesso; invece, se l'argomento da stampare occupa più caratteri di `width`, non si avranno effetti visibili. Tale opzione può essere inserita **direttamente come valore numerico** (ad esempio, **`%3d`** formatta un intero che dovrà occupare almeno 3 caratteri) o **mediante un asterisco** (**`*`**) **e un ulteriore argomento nella `value-list`** (ad esempio, `printf("%*d", 3, 10)` stampa il numero 10 utilizzando uno spazio minimo di 3 caratteri).

L'opzione **`.precision`** permette di **indicare un limite massimo alla "precisione" dell'argomento formattato dal segnaposto**. Cosa si intende con "precisione" dipende dal tipo di dato, ad esempio:
- per i **numeri in virgola mobile** (indicati con `f` o `e`), il valore `precision` indica **la lunghezza massima di cifre decimali dopo cui arrotondare**;
- per le **stringhe** (indicate con `s`), il valore `precision` indica il **numero massimo di caratteri stampati** della stringa considerata (superata tale lunghezza, la stringa viene troncata).

Come per l'opzione `width`, l'opzione `precision` si può indicare o **direttamente come valore numerico** o **mediante un asterisco e un ulteriore argomento** (in entrambi i casi, si ricordi di precedere il valore numerico o l'asterisco con un punto fermo **`.`**).

Infine, l'opzione **`length`** permette di **specificare le dimensioni effettive in memoria dell'argomento**, e alla funzione `printf` di leggere correttamente la dimensione di interi più lunghi o più corti di un tipico `int`. Ad esempio:
- inserire **`hh`** lascia intendere a `printf` di **aspettarsi un numero della dimensione di un `char`** (che verrà, poi, promosso a `int`);
- inserire **`h`** lascia intendere a `printf` di **aspettarsi un numero della dimensione di un `short`** (che verrà, poi, promosso a `int`).

Oltre ai segnaposto, un'altra componente speciale ma molto potente nel contesto della funzione `printf` sono le cosiddette "**escape sequences**", o **sequenze di escape**, che si rivelano utili soprattutto per la **formattazione della stringa di testo da stampare** e per l'**inserimento all'interno della stringa di caratteri speciali**. Le escape sequences iniziano sempre con un **backslash** (**`\`**) **seguito da uno o più caratteri**, e le principali sono:
- **`\a`**, che impone al sistema di **emettere un suono di avviso** contestualmente alla stampa;
- **`\b`**, che equivale a **spostare il cursore indietro di una posizione**, cancellando l'ultimo carattere stampato o sovrascrivendolo con il successivo;
- **`\f`**, che permette di **forzare l'avanzamento alla pagina successiva** (si utilizzava su sistemi e stampanti vecchie, in sistemi moderni è raramente utilizzato e spesso produce output imprevisti);
- **`\n`**, che equivale a **spostare il cursore all'inizio della riga successiva**, andando sostanzialmente "**a capo**"; 
- **`\r`**, che equivale a **spostare il cursore all'inizio della riga corrente** senza andare a capo, dunque il testo successivo andrà a sovrascrivere quello già scritto sulla stessa riga;
- **`\t`**, che equivale a **inserire uno spazio di tabulazione orizzontale** (di solito equivale a 4 o 8 spazi);
- **`\v`**, che equivale a **inserire uno spazio di tabulazione verticale** (in sistemi moderni viene raramente utilizzato e tende a produrre output imprevisti);
- **`\\`**, che permette di **stampare un backslash** (**`\`**);
- **`\'`**, che permette di **stampare una singola virgoletta** (**`'`**);
- **`\"`**, che permette di **stampare una singola doppia virgoletta** (**`"`**);
- **`\?`**, che permette di **stampare un punto interrogativo** (**`?`**) (veniva utilizzato su sistemi più vecchi, ormai è raramente necessario);
- **`\ooo`**, che permette di **rappresentare un carattere usando fino a tre cifre in base ottale** (ad esempio, dato che l'ottale `101` corrisponde ad `A`, stampare `\101` equivale a stampare `A`);
- **`\xhh`**, che permette di **rappresentare un carattere usando fino a due cifre in base esadecimale** (ad esempio, dato che l'esadecimale `x41` corrisponde ad `A`, stampare `\x41` equivale a stampare `A`);
- **`\0`**, che rappresenta il **carattere con valore ASCII `0`**, spesso utilizzato come valore nullo o come [[SO2_04 - C#Stringhe|carattere di fine stringa]].

##### Input

Per **leggere dati in input**, il metodo principale è la funzione **`scanf`**, la cui chiamata assume la seguente forma:

```
scanf(format-string, address-list)
```

A differenza della funzione `printf`, all'interno della quale il parametro `format-string` contiene nella stragrande maggioranza caratteri "costanti", in una chiamata a `scanf` ci si aspetta che questo parametro contenga **esclusivamente o quasi esclusivamente segnaposto**: questo perché la funzione si aspetta che l'utente digiti esattamente i caratteri specificati nella `format-string`, e se l'input dell'utente non combacerà perfettamente con la `format-string`, l'esecuzione fallirà. Ad esempio, se si prova ad eseguire:

```
scanf("Numero: %d", &n);
```

per fare in modo che la funzione memorizzi effettivamente nella variabile `n` il numero inserito dall'utente, quest'ultimo dovrà scrivere esattamente "Numero: " e poi il numero che vuole inserire. L'unica eccezione a questa regola sono gli **spazi vuoti**, gli **a capi** (**`\n`**) e le **tabulazioni** (**`\t`**): se si inseriscono questi spazi o caratteri speciali nella `format-string`, si sta sostanzialmente dicendo alla funzione di ignorare qualsiasi numero di spazi vuoti finché non si trova il prossimo carattere utile.

Anche per `scanf` vale la maggior parte dei **segnaposto** che abbiamo visto nel [[SO2_04 - C#Output|paragrafo precedente]]. Ne ricordiamo comunque i principali:
- **`%d`**, che permette di leggere in input un **numero intero decimale con segno**;
- **`%i`**, che permette di leggere in input un **numero intero con segno**, come `%d`, ma in modo più "intelligente" distinguendo tra diverse basi numeriche (se il numero inizia per **`0x`**, allora verrà letto un **numero esadecimale**, mentre se il numero inizia per **`0`**, allora verrà letto un **numero ottale**);
- **`%u`**, che permette di leggere in input un **numero intero senza segno**;
- **`%f`**, che permette di leggere in input un **numero reale di tipo `float`**;
- **`%lf`**, che permette di leggere in input un **numero reale di tipo `double`**;
- **`%g`**, **`%G`** e **`%e`**, che permettono di leggere in input un **numero reale scritto in notazione scientifica**;
- **`%x`** e **`%X`**, che permettono di leggere in input un **numero intero esadecimale**;
- **`%o`**, che permette di leggere in input un **numero intero ottale**;
- **`%c`**, che permette di leggere in input un **singolo carattere**;
- **`%s`**, che permette di leggere in input una **stringa di testo** (si ferma automaticamente non appena viene letto uno spazio vuoto, una tabulazione o un a capo);
- **`%n`**, che non legge nulla dalla tastiera ma che **scrive nella variabile intera di destinazione quanti caratteri sono stati processati da `scanf` fino a quel momento**.

Similmente a `printf`, anche i segnaposto in `scanf` accettano dei **modificatori di lunghezza** (coincidono con le opzioni `length`), così come un campo per la **lunghezza massima dell'input** (coincide con l'opzione `.precision`). 

Ora, passando al parametro **`address-list`**, come suggerisce il nome esso consisterà in un **elenco degli indirizzi di memoria in cui memorizzare i valori ricevuti in input**: si tratta a tutti gli effetti di un elenco di [[SO2_04 - C#Puntatori|puntatori]], ciascuno separato da una virgola (`,`).

La funzione `scanf` **restituisce il numero di valori letti in input**.

## Variabili e operatori

In C, come in qualsiasi altro linguaggio di programmazione, le **variabili** sono sostanzialmente dei **contenitori di dati**, che permettono di identificare e manipolare comodamente questi ultimi. Concretamente, una variabile è una **locazione di memoria riservata a un determinato dato**; il valore della variabile, dunque il valore memorizzato in tale locazione di memoria, potrà essere modificato quante volte si vuole (a meno che non si definisca la variabile come costante), ma **una variabile potrà sempre contenere un solo valore alla volta**.

##### Identificatori delle variabili

Ogni variabile ha un suo **identificatore**, ossia un **nome univoco** per identificare precisamente tale variabile all'interno del codice.

Nello scegliere un identificatore per una variabile, si dovrà fare attenzione ad alcune **regole**:
- il **primo carattere** di un identificatore deve essere o una **lettera** o un **underscore** (**`_`**), e può essere seguito solamente da una qualsiasi lettera, numero o underscore;
- gli identificatori sono **case-sensitive**, dunque lettere maiuscole e minuscole sono distinte;
- **non sono consentite virgole o spazi vuoti** all'interno di un identificatore;
- **non si possono utilizzare keyword di C** (ad esempio `if`, `while`, `return`, ecc. ecc.) come identificatori;
- gli identificatori hanno una **lunghezza massima di 31 caratteri**.

Oltre alle regole appena stabilite, è ovviamente buona pratica **scegliere identificatori chiari e descrittivi** relativamente al contenuto e allo scopo delle variabili in questione.
___
##### Dichiarazione e tipi di dati

Avendo stabilito delle regole per scegliere l'[[SO2_04 - C#Identificatori delle variabili|identificatore]] di una variabile, siamo pronti per utilizzarle. Il primissimo passo del **ciclo di vita di una variabile** è la sua "**dichiarazione**": in parole povere, in questo passaggio si va a "creare" la variabile, o meglio la locazione di memoria associata ad essa, specificandone il nome da utilizzare per riferirsi ad essa (dunque l'**identificatore**) ma anche il **tipo di dato** che sarà contenuto nella variabile (ciò va fatto sia per allocare un ammontare accurato di memoria, sia per dare maggiori informazioni al compilatore su come gestire la variabile). Dunque, una tipica dichiarazione di variabile assume la seguente forma:

```
optional_modifier data_type name;
```

dove **`data_type`** è proprio il tipo del dato da inserire nella variabile, **`name`** è l'identificatore associato alla variabile (se si vogliono dichiarare più variabili dello stesso tipo contemporaneamente, è possibile inserire più identificatori separati da virgole), e **`optional_modifier`** è, come suggerisce il nome, un **modificatore opzionale**, che si può aggiungere per specificare meglio determinate caratteristiche della variabile. I principali modificatori che si possono inserire sono:
- **`signed`**, che impone a eventuali variabili numeriche di utilizzare il primo bit per memorizzare il segno del numero, seguendo il sistema del complemento a 2 (è il comportamento di default, ad esempio, per le variabili intere);
- **`unsigned`**, che impone a eventuali variabili numeriche di memorizzare esclusivamente numeri positivi, utilizzando dunque tutti i bit disponibili per il valore numerico e ignorando così il sistema del complemento a 2 (in questo modo, si raddoppia il limite massimo positivo raggiungibile);
- **`short`**, utilizzato tipicamente con valori numerici di tipo `int`, impone alla variabile di occupare meno memoria rispetto a un `int` standard (se possibile); tipicamente, uno `short int` occupa 2 byte di memoria;
- **`long`**, utilizzato tipicamente con valori numerici di tipo `int` e `double`, impone alla variabile di occupare uno spazio maggiore o uguale rispetto a un `int` o a un `double` standard (se possibile); tipicamente, un `long int` occupa 4 o 8 byte a seconda del sistema, mentre un `long double` occupa 10 o 16 byte;
- **`const`**, abbreviazione di "constant", che rende una variabile una "costante", ossia un valore di sola lettura che, una volta assegnato, non potrà essere modificato o sostituito da istruzioni successive.

Dunque, abbiamo stabilito che per poter utilizzare una variabile si dovrà prima stabilirne il tipo, tra le altre cose. Ma **quali sono i possibili tipi di dato** all'interno di C?

Partiamo dai **tipi di dato numerici**, che possiamo innanzitutto dividere in **numeri interi** e **numeri reali** (con la virgola). A loro volta, i numeri interi si dividono in:
- **`short`**, o **`short int`**, che rappresenta un intero utilizzando **2 byte** di memoria;
- **`int`**, che rappresenta un intero utilizzando **4 byte** di memoria;
- **`long`**, che rappresenta un intero utilizzando **8 byte** di memoria.

Tra i reali, invece, troviamo:
- **`float`**, che rappresenta un numero con la virgola utilizzando **4 byte** di memoria (corrisponde allo standard **IEEE-754 single-precision**);
- **`double`**, che rappresenta un numero con la virgola utilizzando **8 byte** di memoria (corrisponde allo standard **IEEE-754 double-precision**);
- **`long double`**, che rappresenta un numero con la virgola utilizzando **10 o 16 byte** di memoria.

Ovviamente, per molti dei tipi numerici appena mostrati sono disponibili variazioni in base all'eventuale utilizzo di modificatori.

C'è poi un tipo che, per certi versi, fa da ponte tra i tipi numerici e le stringhe di testo: il tipo **`char`**. Si tratta di un tipo che, nel concreto, occupa **1 byte** di memoria e serve per memorizzare un **qualsiasi carattere della codifica ASCII**; analizzandolo più da vicino, però, si osserva che in realtà una variabile di tipo `char` non fa altro che memorizzare un intero compreso tra $0$ e $127$, ed è in base a tale intero che viene effettuata l'associazione con un carattere specifico (proprio per questa "doppia natura" del tipo `char`, è possibile utilizzare una variabile del genere sia come numero che come carattere di testo).

Un altro tipo di dato molto importante è il **valore booleano**, un valore che può essere solo vero (**`true`**) o falso (**`false`**). In C, un valore booleano è principalmente rappresentato in due modi:
- dal tipo **`_Bool`**, che può memorizzare solo `0` o `1`, per cui se si prova ad assegnare a una variabile di tale tipo un qualsiasi valore diverso da `0` esso verrà automaticamente convertito a `1`;
- dal tipo **`bool`**, che memorizza effettivamente i simboli `true` o `false` ma che richiede l'inclusione della libreria **`stdbool.h`**.

In generale, poi, in qualsiasi espressione logica un risultato pari a `0` viene interpretato come `false` dal compilatore, mentre un risultato diverso da `0` è tipicamente interpretato come `true`.

**Il tipo di una variabile non è sempre fisso**: infatti, a seconda del contesto e di come viene utilizzato, un valore può cambiare tipo. Questa **conversione di tipo** viene detta "**casting**", e può avvenire principalmente in due modi:
- **casting implicito**, detto anche "**promozione**";
- **casting esplicito**.

Il **casting implicito**, come suggerisce il nome, avviene **in modo automatico** quando il compilatore decide di **convertire un dato in un tipo "più grande"** per non perdere informazioni, o anche quando si assegna il valore di una determinata variabile a una variabile di tipo diverso. Un esempio molto comune di casting implicito può essere visto effettuando un'operazione matematica tra un `int` e un `double` (in questo caso, il compilatore promuoverà automaticamente il valore di tipo `int` a un `double`, e solo in seguito effettuerà l'operazione); lo stesso meccanismo viene applicato eseguendo, ad esempio, il seguente codice:

```
int num = 10;
double num1 = num;
```

dato che `num`, contenente un valore di tipo `int`, viene assegnato a una variabile di tipo "più grande", ossia `double`, e perciò il compilatore effettua una promozione convertendo `10` in `10.0`.

Il **casting esplicito**, invece, avviene come suggerisce il nome **in modo esplicito** a livello di codice, nel senso che è il programmatore a forzare la conversione. Per fare ciò, si dovrà specificare il tipo di dato desiderato tra parentesi tonde (**`()`**) subito prima del valore o della variabile in questione, seguendo la seguente sintassi:

```
(data_type) expression;
```

Seppur meno comune del casting implicito, anche quello esplicito può essere utile: ad esempio, se eseguire il seguente codice:

```
int a = 11;
int b = 2;

double res = a / b;
```

fornisce un risultato non accurato, dato che essendo `a` e `b` entrambi operatori interi il compilatore eseguirà una divisione intera, eseguire invece il seguente codice:

```
int a = 11;
int b = 2;

double res = (double) a / b;
```

fornirà invece il risultato corretto, dato che il compilatore convertirà prima `a` in un valore `double`, e di conseguenza procederà a promuovere `b` e a calcolare la divisione in modo accurato.

Ma **cosa succede se convertiamo** (implicitamente o esplicitamente) **un numero reale in un numero intero**? In questi casi, non avverrà un arrotondamento matematico, come ci si potrebbe aspettare, ma piuttosto un **troncamento**: dunque, qualsiasi sia la parte decimale del numero considerato, e sia quest'ultimo positivo o negativo, fare ciò non farà altro che **eliminare la parte decimale**, conservando unicamente quella intera.

##### Assegnazione di valori alle variabili

Per poter utilizzare una variabile non basta [[SO2_04 - C#Dichiarazione e tipi di dati|dichiararla]]: infatti, la sola dichiarazione di una variabile non fa altro che allocare uno spazio di memoria ad essa, ma il suo valore effettivo rimane di fatto indeterminato. Sarà, dunque, necessario **assegnare un valore alla variabile**; ciò può essere fatto in vari modi, ad esempio:
- **contemporaneamente alla dichiarazione**;
- **in seguito alla dichiarazione**;
- **leggendo un valore in [[SO2_04 - C#Input e output|input]]**.

Tipicamente, per assegnare un valore alla variabile dal codice, dunque non considerando l'input dell'utente, si utilizza l'operatore binario **`=`**, nel modo seguente:

```
l_value = r_value;
```

dove `l_value` è tipicamente una variabile o un puntatore, mentre `r_value` è il valore che si vuole assegnare ad esso, o eventualmente l'espressione il cui risultato va assegnato a `l_value`. Dunque, se volessimo ad esempio dichiarare una variabile `x` di tipo `int`, e assegnarle il valore `3`, potremmo scrivere:

```
int x = 3;
```

o, alternativamente:

```
int x;

...

x = 3;
```

##### Operatori

Come detto nel [[SO2_04 - C#Assegnazione di valori alle variabili|paragrafo precedente]], a una variabile può anche essere assegnato il risultato di un'espressione: all'interno di C, infatti, troviamo numerosi **operatori** che permettono di scrivere espressioni più o meno complesse. Possiamo dividere gli operatori in categorie nel modo seguente:
- operatori **aritmetici**, come **`+`**, **`-`**, **`*`**, **`/`** o **`%`**;
- operatori **relazionali**, come **`==`**, **`!=`**, **`<`**, **`<=`**, **`>`**, o **`>=`**;
- operatori **logici**, come **`!`**, **`&&`** o **`||`**;
- operatori **bitwise** (o **bit a bit**), come **`&`**, **`|`**, **`~`** o **`^`**;
- operatori di **shift** (o di **scorrimento**), come **`<<`** o **`>>`**.

Gli operatori aritmetici sono, ovviamente, corrispondenti alle **operazioni matematiche più comuni** (`+` per l'addizione, `-` per la sottrazione, `*` per la moltiplicazione, `/` per la divisione e `%` per il resto della divisione). Naturalmente, c'è un **ordine di precedenza** nella valutazione dei vari operatori:
1. operazioni incluse tra **parentesi tonde** (**`()`**), e in caso di parentesi annidate vengono valutate prima quelle più interne;
2. moltiplicazione (`*`), divisione (`/`) e resto della divisione (`%`), e in caso ce ne siano di più vengono valutate da sinistra verso destra;
3. addizione (`+`) e sottrazione (`-`), e in caso ce ne siano di più vengono valutate da sinistra verso destra.

Un approfondimento sull'operatore di divisione (`/`): **se si effettua una divisione tra due numeri interi, verrà effettuata una divisione intera** anche se, matematicamente, il risultato dovrebbe avere una parte decimale; inoltre, **se si effettua una divisione tra due tipi numerici diversi, il risultato sarà del tipo più "grande"** tra i due (ad esempio, se si effettua una divisione tra un `int` e un `double`, l'intero verrà considerato come `double` e lo stesso tipo sarà attribuito anche al risultato).

Spesso, con alcuni operatori, è possibile scrivere delle **abbreviazioni**, perfettamente equivalenti alla scrittura estesa a livello di funzionamento, ad esempio:
- `var = var + n` può essere abbreviato in **`var += n`**;
- `var = var - n` può essere abbreviato in **`var -= n`**;
- `var = var * n` può essere abbreviato in **`var *= n`**;
- `var = var / n` può essere abbreviato in **`var /= n`**;
- `var = var % n` può essere abbreviato in **`var %= n`**;
- `var = var & n` può essere abbreviato in **`var &= n`**;
- `var = var | n` può essere abbreviato in **`var |= n`**;
- `var = var ^ n` può essere abbreviato in **`var ^= n`**;
- `var = var << n` può essere abbreviato in **`var <<= n`**;
- `var = var >> n` può essere abbreviato in **`var >>= n`**.

Un'abbreviazione particolare ma anche molto utile è quella legata agli **incrementi** (aumento del valore di una variabile di un'unità) e ai **decrementi** (diminuzione del valore di una variabile di un'unità): infatti, piuttosto che scrivere `var = var + 1` è possibile scrivere **`var++`**, e analogamente piuttosto che scrivere `var = var - 1` è possibile scrivere **`var--`**. Va specificata, su questo argomento, una **differenza tra post-incremento** (o decremento) **e pre-incremento** (o decremento):
- il **post-incremento** (**`var++`**) prima legge il valore di `var` e poi lo incrementa di 1;
- il **pre-incremento** (**`++var`**) prima incrementa di 1 il valore di `var` e poi lo legge.

## Istruzioni di controllo

Le **istruzioni di controllo** sono istruzioni che, come suggerisce il nome, permettono di **controllare ed, eventualmente, redirezionare il flusso di esecuzione** del programma. In base alla struttura e allo scopo, possiamo categorizzare le varie istruzioni di controllo in:
- **istruzioni di iterazione**, ossia istruzioni che permettono di eseguire ripetutamente uno stesso blocco di codice (in questo paragrafo, approfondiremo il `while` loop, il `for` loop e il `do-while` loop);
- **istruzioni condizionali**, ossia istruzioni che permettono di eseguire un blocco di codice piuttosto che un altro sulla base di una certa condizione (in questo paragrafo, approfondiremo l'`if` statement);
- **istruzioni di salto**, ossia istruzioni che permettono di saltare l'esecuzione di certe parti del codice (in questo paragrafo, approfondiremo le istruzioni `break` e `continue`).

##### Il `while` loop

Il **`while` loop** è un ciclo che dipende da una **condizione valutata prima di entrare nel loop stesso**: finché la condizione valutata è vera (ossia, valutarla restituisce un valore `true`), viene rieseguito il corpo di codice e in seguito rivalutata la condizione. Generalmente, un `while` loop assume la seguente forma:

```
while (condition)
{
	basic block
}
```

dove **`condition`** è la condizione che viene valutata, mentre il `basic block` è un blocco di codice qualsiasi. Banalmente, **nel corpo di codice deve accadere qualcosa che può eventualmente "cambiare" la condizione**, altrimenti il loop procederebbe all'infinito. L'iterazione del ciclo termina quando **`condition`** diventa falsa o, nel caso in cui il blocco di codice la includa, all'esecuzione dell'[[SO2_04 - C#Le istruzioni di salto|istruzione di salto]] `break` (la approfondiremo in seguito).

Come detto, **ci deve essere almeno una variabile il cui valore controlla la condizione del ciclo**, altrimenti tale condizione risulterebbe essere sempre vera (finendo in un ciclo infinito) o sempre falsa (impedendo l'esecuzione del ciclo): tale variabile viene detta **LCV**, o "**Loop Control Variable**", e tipicamente, in relazione a un ciclo `while`, deve rispettare le seguenti proprietà:
- devono essere **dichiarate e inizializzate prima del ciclo**;
- devono essere **lette nella condizione del ciclo**;
- devono essere **aggiornate nel corpo del ciclo**, in modo che prima o poi la condizione da vera diventi falsa.
___
##### Il `for` loop

Il **`for` loop**, similmente al [[SO2_04 - C#Il `while` loop|while]], è un ciclo che dipende da una **condizione valutata prima di entrare nel loop stesso**; la differenza, però, sta nel tipo di condizione valutata. Infatti, mentre il `while` loop si basa su una condizione booleana costituita da una qualsiasi espressione, il `for` loop **basa il numero di iterazioni interamente sul valore della LCV**, che varia in modo costante e prevedibile ad ogni iterazione del ciclo in questione. Ciò vuol dire che, a differenza del `while` loop, **un `for` loop eseguirà sempre un numero di iterazioni noto a priori** (tecnicamente, si può prevedere a grandi linee anche il numero di iterazioni che eseguirà un `while` loop, tuttavia ciò risulta più complesso e spesso risulta comunque in una previsione "parametrizzata", mentre con i `for` loops il numero di iterazioni è sempre noto).

Generalmente, un `for` loop assume la seguente forma:

```
for (initialization; test; increment)
{
	basic block
}
```

dove **`initialization`** corrisponderà all'**inizializzazione della LCV** a un certo valore, **`test`** corrisponderà alla **condizione sulla LCV che dovrà essere vera per continuare l'iterazione del ciclo**, e **`increment`** è la **variazione che subisce la LCV dopo ogni iterazione del ciclo**. Dunque, la LCV parte da un certo valore, e quest'ultimo cambia in modo prevedibile a ogni iterazione; quando il valore della LCV sarà tale per cui la condizione specificata non sarà più rispettata, allora il ciclo terminerà la sua esecuzione.

C'è un dettaglio su cui è opportuno soffermarsi: spesso, in altri linguaggi, la LCV non viene solo inizializzata ma anche dichiarata all'interno del ciclo `for`, rendendola a tutti gli effetti una variabile locale al ciclo stesso; ad esempio, in C++ tipicamente si vedrà un ciclo `for` nella seguente forma:

```
for (int i = 0; i < 10; i++)
{

}
```

In C, invece, **a volte si dichiara la LCV fuori dal ciclo**, prima che esso venga eseguito. Ad esempio, il ciclo `for` appena visto potrebbe apparire nella seguente versione:

```
int i;
for (i = 0; i < 10; i++)
{

}
```

Ciò è dovuto più che altro a motivi storici: prima del 1999, lo standard ufficiale del linguaggio C (noto come **ANSI C** o anche **C89**) non permetteva in alcun modo la sintassi vista nel primo esempio, dato che imponeva che tutte le variabili dovevano essere dichiarate all'inizio del codice (non si trattava solo di una convenzione, ma di una vera e propria restrizione che avrebbe impedito la compilazione del codice sorgente); la sintassi "condensata", che vediamo ad oggi in molti linguaggi, è stata introdotta solo nello standard **C99**. 

Dunque, seppur C sia perfettamente in grado di riconoscere cicli `for` scritti dichiarando la LCV all'interno degli stessi, a volte si mantiene il vecchio metodo per una questione di retro-compatibilità, o in alcuni contesti in cui gli standard dal C99 in poi non sono proprio supportati (ad esempio in alcuni sistemi embedded). Ad ogni modo, a meno che non sia necessario conoscere il valore del contatore dopo che il ciclo considerato è terminato, **ad oggi è preferibile dichiarare la LCV all'interno del ciclo**.
___
##### Il `do-while` loop

Il **`do-while` loop** è una semplice **variazione del ciclo [[SO2_04 - C#Il `while` loop|while]]** che abbiamo visto un paio di paragrafi fa: la differenza principale è che **la condizione viene valutata dopo aver eseguito il blocco di codice**. Generalmente, un `do-while` loop assume la seguente forma:

```
do 
{
	basic block
	
} while (condition);
```

Questa diversa struttura del ciclo garantisce che **il corpo del ciclo venga eseguito almeno una volta**, caratteristica che può rivelarsi utile in certe situazioni (ad esempio quando si vuole ricevere un input dall'utente e si vuole continuare a chiederlo finché l'utente non inserirà un valore valido).
___
##### Le istruzioni di salto

Come anticipato, le **istruzioni di salto** sono istruzioni utilizzate per **saltare l'esecuzione di certe parti del codice**. In particolare, andremo ad analizzare il funzionamento delle seguenti istruzioni di salto:
- **`break`**;
- **`continue`**.

L'istruzione **`break`** può essere utilizzata per **uscire anticipatamente da un qualsiasi ciclo o da uno `switch` statement**, un tipo particolare di istruzione condizionale. Insomma, se un'istruzione `break` viene eseguita all'interno di un certo blocco di codice, essa porterà al termine dell'esecuzione del costrutto in cui tale istruzione si trova (indipendentemente dall'eventuale valore della condizione o della LCV). Di seguito, un esempio di utilizzo dell'istruzione `break`:

```
for (int i = 1; i <= 5; i++) 
{ 
	if (i == 3) break;     // arrivato i a 3, il ciclo termina
	printf("%d ", i); 
}
```

L'istruzione **`continue`**, invece, può essere utilizzata per **forzare il passaggio alla prossima iterazione di un ciclo**, senza eseguire le istruzioni rimaste nel blocco di codice in questione. Di seguito, un esempio di utilizzo dell'istruzione `continue`:

```
for (int i = 1; i <= 5; i++) 
{ 
	if (i == 3) continue; // arrivato i a 3, salta alla prossima iterazione 
	printf("%d ", i); 
}
```
___
##### L'`if` statement

L'**`if` statement** è l'istruzione condizionale più comune e semplice nella programmazione, non solo in C. Concretamente, equivale a dire "se succede… allora fai…": si tratta, più formalmente, di **valutare una condizione** e di **eseguire un blocco di codice solo se tale condizione è verificata**. Come è facilmente intuibile, per implementare tale funzionamento la condizione da valutare dovrà essere, concretamente, un'**espressione [[SO2_04 - C#Dichiarazione e tipi di dati|booleana]]**.

Generalmente, un `if` statement assume la seguente forma:

```
if (boolean_expression)
{
	// BLOCCO DI CODICE
}
```

Molto spesso, poi, può essere comodo anche prevedere un comportamento nel caso in cui la condizione `boolean_expression` non si verifichi (tornando all'esempio di prima, si immagini di dire "se succede… allora fai…, altrimenti fai…"): è in tal caso che si può utilizzare, abbinato all'`if` statement, l'**`else` statement**. Scrivendo, dunque, il seguente codice:

```
if (boolean_expression)
{
	// BLOCCO DI CODICE
} 
else
{
	// BLOCCO DI CODICE
}
```

prevediamo che, se `boolean_expression` viene valutata come `true`, allora viene eseguito il blocco di codice contenuto nella prima coppia di parentesi graffe, altrimenti (dunque se `boolean_expression` viene valutata come `false`) viene eseguito il blocco di codice contenuto nella seconda coppia di parentesi graffe. Nonostante sia spesso molto utile, si precisa che **inserire un `else` statement è totalmente opzionale**.

E se volessimo **valutare più di due alternative**? In tal caso, ampliamo ancora di più il nostro costrutto introducendo gli **`else if` statements**, che sono una sorta di "ponte" tra `if` e `else`: come l'`if`, anche gli `else if` valutano un'espressione booleana ed eseguono il loro blocco di codice se quest'ultima si rivela `true`; come l'`else`, poi, gli `else if` vengono presi in considerazione solamente se tutte le alternative precedenti ad essi (siano esse gestite tramite un `if` o altri `else if`) valutano come `false` la propria condizione. Un esempio generico di quella che può essere chiamata "**`if`-`else if` ladder**" è il seguente:

```
if (espressione_booleana_1)
{
	// BLOCCO DI CODICE
}
else if (espressione_booleana_2)
{
	// BLOCCO DI CODICE
}
else if (espressione_booleana_3)
{
	// BLOCCO DI CODICE
}
else
{
	// BLOCCO DI CODICE
}
```

## Array

Gli **array** sono un tipo di dato strutturato, che possiamo vedere come un **insieme indicizzato di valori** (per certi versi, possiamo vederla come una serie di variabili). I valori contenuti in un array hanno le seguenti caratteristiche:
- i valori devono essere dello **stesso tipo**;
- occupano tutti una **regione contigua di memoria**;
- la **dimensione dell'array**, dunque il numero di valori contenuti al suo interno, **rimane invariata** nel corso dell'esecuzione.

Come anticipato, gli array sono un insieme "**indicizzato**" di valori: ciò vuol dire che, oltre all'array come insieme, si può facilmente **accedere anche ai singoli elementi contenuti nell'array** conoscendo la loro posizione al suo interno, posizione che viene detta proprio "**indice**". Gli indici vengono assegnati agli elementi contenuti in un array in ordine crescente e partendo da `0`: ciò vuol dire che il primo elemento dell'array avrà indice `0`, che il secondo elemento avrà indice `1`, e così via fino ad arrivare all'ultimo elemento, che avrà indice pari alla dimensione dell'array decrementata di 1.

##### Dichiarazione di un array

Come per una qualsiasi [[SO2_04 - C#Variabili e operatori|variabile]], per utilizzare un array sarà necessario prima di tutto dichiararlo, in modo che possa venire allocata la memoria destinata ad essi (tale memoria viene interamente allocata quando il programma viene caricato in memoria). La **dichiarazione di un array** segue la seguente sintassi:

```
data_type name[dim];
```

dove **`data_type`** è il **tipo degli elementi** che verranno contenuti nell'array, **`name`** è l'**identificatore** dell'array (un trucchetto utile da ricordare è che, per il compilatore, **riferirsi all'identificatore dell'array equivale a utilizzare un puntatore all'indirizzo di memoria contenente il primo elemento dell'array**, ossia quello con indice `0`) e **`dim`** è la **dimensione** dell'array, ossia il numero di elementi che potrà essere contenuto al suo interno. In particolare, nel dichiarare la dimensione dell'array (che naturalmente può essere un qualsiasi numero intero positivo) conviene adottare un certo comportamento: spesso, piuttosto che inserire effettivamente dei numeri senza un significato meglio precisato, conviene **creare una costante** o utilizzare una **direttiva per il preprocessore**, in modo da associare al numero in questione un nome preciso ed esplicativo.
___
##### Inizializzazione di un array

Una volta dichiarato, un array dovrà essere **inizializzato**, ossia riempito di valori effettivi. Ci sono vari modi in cui può avvenire un'inizializzazione di array. Il primo che andremo a vedere, forse il più semplice, è l'**inizializzazione completa e diretta dal codice**, che avviene specificando proprio nel codice i precisi elementi che dovranno essere inseriti nell'array, nell'ordine esatto in cui sono specificati. Ad esempio:

```
int voti[5] = {7, 9, 8, 7, 10};
```

È possibile anche effettuare un'**inizializzazione parziale dell'array**, specificando cioè solo una parte degli elementi da memorizzare al suo interno. Ad esempio, la seguente istruzione:

```
int voti[5] = {7, 9};
```

assegnerà all'indice `0` il valore 7, all'indice `1` il valore 9, mentre lascerà che le altre posizioni vengano riempite da **valori di default**, variabili in base al tipo degli elementi contenuti nell'array (`0` per gli interi, `0.0` per i reali, e così via). Ancora, è possibile anche **inizializzare tutti gli elementi di un array di valori numerici a `0`** molto semplicemente, con la seguente assegnazione:

```
int voti[5] = {};
```

Infine, è possibile anche **far inizializzare l'array all'utente durante l'esecuzione del programma** in vari modi, a seconda delle esigenze e anche dello standard a cui si sceglie di aderire. Il metodo più semplice è l'**inizializzazione di un array con un numero fisso di elementi**: in questo scenario, la dimensione dell'array, e dunque anche il numero di elementi che verranno chiesti all'utente, sono decisi a priori durante la scrittura del codice, e l'array viene inizializzato tramite un semplice [[SO2_04 - C#Il `for` loop|ciclo for]]. Ad esempio:

```
#define DIM 5

int main()
{
	int arr[DIM];
	
	for (int i = 0; i < DIM; i++)
	{
		printf("Inserire un numero intero: ");
		scanf("%d", &arr[i]);
	}
	
	return 0;
}
```

Un'altra alternativa, leggermente più dinamica, è **prevedere una dimensione massima per l'array ma far scegliere all'utente quanti elementi inserire**: anche stavolta l'array verrà dichiarato a priori, ma la differenza è che la sua dimensione sarà una sorta di "limite massimo" deciso dal programmatore, mentre all'utente sarà data libera scelta su quanti elementi inserire nell'array, a patto che tale quantità non superi quella massima imposta dal programmatore. Ad esempio:

```
#define MAX 100

int main()
{
	int arr[MAX];
	int n;
	
	printf("Quanti numeri si vogliono inserire (max: %d)? ", MAX);
	scanf("%d", &n);
	
	if (n <= 0 || n > MAX)
	{
		printf("Numero non valido!\n");
		return 1;
	}
	
	for (int i = 0; i < n; i++)
	{
		printf("Inserire un numero: ");
		scanf("%d", &arr[i]);
	}

	return 0;
}
```

Ancora, è possibile **chiedere all'utente quanti elementi vuole inserire e dichiarare e inizializzare un array avente esattamente tale dimensione**. Ad esempio:

```
int main()
{
	int n;
	
	printf("Quanti numeri si vogliono inserire? ");
	scanf("%d", &n);
	
	if (n <= 0)
	{
		printf("Devi inserire almeno un numero!\n");
		return 1;
	}
	
	int arr[n];
	
	for (int i = 0; i < n; i++)
	{
		printf("Inserire un numero: ");
		scanf("%d", &arr[i]);
	}
	
	return 0;
}
```

Tuttavia, quest'ultimo approccio può portare a risultati problematici in certe situazioni, ossia quelle in cui l'utente inserisce una dimensione `n` molto grande: infatti, **le variabili e gli array dichiarati normalmente vengono memorizzati nello stack**, un'area di memoria molto veloce ma anche molto piccola (su molti OS moderni, lo stack si limita a 8 MB per un intero programma), e se l'utente inserisce una dimensione `n` troppo grande si può incorrere in un problema chiamato "**stack overflow**", ossia nel tentativo di occupare più memoria di quella disponibile in totale, il che porterebbe a un crash del programma; per evitare ciò, la soluzione migliore è l'utilizzo dell'**[[SO2_04 - C#Allocazione dinamica di memoria|allocazione dinamica di memoria]]**, argomento che approfondiremo in seguito.
___
##### Array multi-dimensionali

Naturalmente, oltre ai classici array "lineari", è possibile lavorare anche con **array multi-dimensionali**. Ad esempio, un **array bidimensionale** non è altro che un array di array, che possiamo vedere come una tabella in cui ogni casella è identificata da una certa riga (l'indice dell'array esterno, che indica a quale degli array accedere) e da una certa colonna (l'indice dell'array interno, che indica a quale elemento dell'array considerato accedere). Per dichiarare un array bidimensionale, si utilizza la seguente sintassi:

```
data_type arr[dim1][dim2];
```

dove **`data_type`** è il tipo degli elementi più interni, ossia di quelli contenuti all'interno degli array interni, **`arr`** è l'identificatore dell'array, **`dim1`** è la dimensione dell'array esterno, ossia il numero di array interni contenuti in quest'ultimo, e **`dim2`** è la dimensione degli array interni.

Seguendo gli stessi ragionamenti, è possibile definire array tridimensionali, quadridimensionali e così via.

## Blocchi

Un **blocco di codice** è una **porzione di codice racchiusa tra parentesi graffe** (**`{}`**). Le variabili definite in un blocco si dicono **"locali" a tale blocco**: ciò vuol dire che tali variabili **vengono allocate solo quando il blocco è eseguito**, **non sono più accessibili dopo che il blocco è stato eseguito**, e nel blocco **vengono "oscurate" eventuali variabili con lo stesso nome definite fuori dal blocco** considerato.

Ad esempio, il seguente codice:

```
int main()
{
	{
		int i = 0;
		printf("i = %d\n", ++i);
	}
	
	printf("i = %d\n", ++i);
	
	return 0;
}
```

è sintatticamente errato, dato che contestualmente alla seconda chiamata a `printf` la variabile `i`, locale al blocco precedente, non è visibile e non può dunque essere utilizzata. Vediamo un altro esempio:

```
int main()
{
	int i = -1;
	
	{
		int i = 0;
		printf("i = %d\n", i++);
	}
	
	printf("i = %d\n", i);

	return 0;
}
```

In questo caso, vediamo un esempio di oscuramento di variabili omonime: nel blocco del `main()` viene dichiarata una variabile `i` e inizializzata a -1; poi, nel blocco seguente, viene dichiarata e inizializzata una nuova variabile `i` locale a tale blocco, dunque il `printf` contenuto in quel blocco stamperà `i = 0`; analogamente, l'incremento `i++` affliggerà la variabile locale e non quella dichiarata nel `main()`, motivo per cui il secondo `printf` stamperà `i = -1`.

## Stringhe

In C, le **stringhe** di testo non sono altro che **[[SO2_04 - C#Array|array]] di [[SO2_04 - C#Dichiarazione e tipi di dati|caratteri]]**: ogni elemento di questo array è dunque un carattere della stringa, e l'ultimo elemento dell'array conterrà sempre il cosiddetto "**carattere di fine stringa**", ossia **`\0`** (detto anche **`NULL`**). Dunque, tecnicamente, l'array di caratteri rappresentante una stringa di testo avrà sempre una lunghezza superiore di un'unità rispetto al numero di caratteri effettivi della stringa.

##### Dichiarazione e inizializzazione di una stringa

Essendo una stringa un array di caratteri, il **tipo di dato** da attribuire a una variabile che vada a contenere una stringa è **`char s[dim]`**, dove `dim` è naturalmente la dimensione massima dell'array, mentre `s` è l'identificatore assegnato alla stringa in questione, o meglio alla variabile che la contiene. Ad esempio, volendo **dichiarare una stringa** con identificatore `stringa` e lunga al massimo 10 caratteri, andremo a scrivere:

```
char stringa[10];
```

Se la dichiarazione è perfettamente analoga a quella di un normale array, lo stesso non si può dire per l'**inizializzazione di una stringa**: tipicamente, non andremo infatti a utilizzare le parentesi graffe (`{}`) come per gli array, ma piuttosto si andrà a **scrivere la stringa tra virgolette (`""`)**, nel modo seguente:

```
char stringa[10] = "Lezione 7";
```

In questo esempio, si noti che la stringa effettiva è lunga 9 caratteri (incluso lo spazio vuoto), ma l'array di caratteri della stringa è comunque completamente riempito grazie al carattere di fine stringa `\0`. È perfettamente possibile, comunque, anche trattare la stringa letteralmente come un array di caratteri, e dunque riempire quest'ultimo con singoli caratteri come nell'esempio seguente:

```
char stringa[10] = {'L', 'e', 'z', 'i', 'o', 'n', 'e', ' ', '7', '\0'};
```

Seppur sia un approccio più scomodo e raro, è equivalente a quello visto in precedenza (se si sceglie di inizializzare una stringa in questo modo, si deve ricordare di inserire al termine dell'array sempre il carattere di fine stringa).

E se assegnassimo una **stringa più piccola della dimensione massima disponibile**? In quel caso, semplicemente, **le posizioni non occupate da caratteri all'interno dell'array conterrebbero dei valori indeterminati**. Ad esempio, nella seguente stringa:

```
char s[10] = "L9 4apr";
```

sono occupate solo le prime 8 posizioni dell'array di caratteri, mentre le ultime due conterranno degli elementi indeterminati (il compilatore leggerà comunque solo i caratteri effettivi, dato che si fermerà appena leggerà il carattere `\0`).

Infine, è possibile anche **dichiarare una stringa senza specificare la dimensione dell'array di caratteri, purché venga anche inizializzata nella stessa istruzione**. Ad esempio, scrivendo:

```
char s[] = "L9 4apr";
```

il compilatore allocherà alla stringa `s` l'ammontare di memoria esattamente necessario per contenerla (in questo caso, 8 byte).

##### Funzioni utili

All'interno del file di intestazione **`string.h`** si trovano molte **funzioni utili per lavorare sulle stringhe**. Per poter accedere a tale file, e quindi alle funzioni contenute al suo interno, si dovrà includere il file di intestazione con una direttiva al preprocessore:

```
#include <string.h>
```

All'interno di `string.h`, tra le funzioni per le stringhe più utili e comuni troviamo:
- **`strlen`**;
- **`strcpy`**;
- **`strcmp`**;
- **`strcat`**;
- **`strncpy`**;
- **`strncat`**;
- **`strncmp`**.

La funzione **`strlen`** viene utilizzata per **calcolare la lunghezza effettiva di una stringa**. In particolare, la lunghezza calcolata dalla funzione non tiene conto del carattere di fine stringa `\0` (considera, perciò, solo i caratteri effettivamente contenuti nella stringa) e di eventuali caratteri indeterminati nell'array di caratteri considerato. Una chiamata alla funzione `strlen` rispetta la seguente **sintassi**:

```
strlen(s)
```

dove `s` è la stringa di cui si vuole calcolare la lunghezza. Il **valore ritornato** dalla funzione è di [[SO2_04 - C#Dichiarazione e tipi di dati|tipo]] **`unsigned int`**, e viene calcolato con una **complessità di $O(n)$**.

La funzione **`strcpy`** viene utilizzata per **copiare una stringa in un'altra**. Una chiamata alla funzione `strcpy` rispetta la seguente sintassi:

```
strcpy(s1, s2)
```

dove `s1` è la stringa di destinazione, dunque quella in cui verrà copiata la stringa, e `s2` è la stringa di sorgente, dunque quella che verrà copiata in `s1` (carattere di fine stringa incluso). Il **valore ritornato** dalla funzione è un **puntatore alla stringa di destinazione**. Nell'utilizzare la funzione `strcpy`, bisognerà fare attenzione all'eventualità di un "**buffer overflow**": la funzione non controlla se, durante la copia, si è già esaurito lo spazio destinato alla stringa di destinazione, dunque se la stringa di sorgente eccede le dimensioni di quest'ultima potrebbero essere sovrascritti dati contenuti in altre variabili.

La funzione **`strcmp`** viene utilizzata per **confrontare due stringhe in base al loro ordine lessicografico**, ossia in base al codice ASCII corrispondente a ciascun carattere. Più nel dettaglio: la funzione inizia confrontando i codici ASCII dei primi caratteri delle due stringhe, e se sono diversi possono avvenire due scenari diversi:
- se il codice ASCII del primo carattere della prima stringa è **maggiore** del codice del primo carattere della seconda, allora la funzione restituisce la **differenza positiva** tra i due codici;
- se il codice ASCII del primo carattere della prima stringa è **minore** del codice del primo carattere della seconda, allora la funzione restituisce la **differenza negativa** tra i due codici.

Invece, se i due caratteri considerati sono uguali, si procede allo stesso modo con la successiva coppia di caratteri, e così via; questa operazione viene ripetuta o finché si trovano due caratteri diversi o finché si trova il carattere di fine stringa (`\0`) in entrambe le stringhe considerato, caso in cui le due stringhe risultano identiche e la funzione restituisce **`0`**. Una chiamata alla funzione `strcmp` rispetta la seguente sintassi:

```
strcmp(s1, s2)
```

dove `s1` e `s2` sono le due stringhe da confrontare.

La funzione **`strcat`** viene utilizzata per **concatenare due stringhe**, ossia "attaccare" una stringa al termine di un'altra. Una chiamata alla funzione `strcat` rispetta la seguente sintassi:

```
strcat(s1, s2)
```

dove `s1` è la stringa di destinazione, dunque la stringa a cui verrà accodata l'altra, mentre `s2` è la stringa di sorgente, che verrà concatenata al termine di `s1`. Nel dettaglio, quello che succede è che il carattere di fine stringa di `s1` viene sovrascritto dal primo carattere di `s2`; in seguito, vengono aggiunti uno per uno gli altri caratteri di `s2`, e infine si aggiunge anche un nuovo carattere di fine stringa. Il **valore ritornato** è dalla funzione è un **puntatore alla stringa di destinazione**. Naturalmente, nel chiamare la funzione su due stringhe, è importante **assicurarsi che la stringa di destinazione possa contenere abbastanza caratteri da ospitare completamente la concatenazione**.

Si può notare che le funzioni appena viste, in particolare `strcpy` e `strcat`, possono avere comportamenti sbagliati o imprevisti se non si fa attenzione alle dimensioni delle stringhe considerate. Per scrivere codice in modo più sicuro, è possibile utilizzare **varianti "sicure" di tali funzioni**, ossia:
- **`strncpy`**;
- **`strncat`**;
- **`strncmp`**.

La funzione **`strncpy`** viene utilizzata per **copiare un certo numero di caratteri di una stringa in un'altra**. Una chiamata alla funzione `strncpy` rispetta la seguente sintassi:

```
strncpy(s1, s2, n)
```

dove `s1` è la stringa di destinazione, dunque quella in cui verrà copiata la stringa, `s2` è la stringa di sorgente, dunque quella che verrà copiata in `s1`, e **`n`** è il numero di caratteri da copiare da `s2` in `s1`. Il **valore ritornato** dalla funzione è un **puntatore alla stringa di destinazione**. In base al valore di `n` in relazione alla lunghezza delle due stringhe, è importante tenere a mente alcuni scenari particolari:
- nel caso in cui **la stringa di sorgente contenga un numero di caratteri maggiore o uguale a `n`**, la funzione non inserirà automaticamente il carattere di fine stringa al termine della stringa di destinazione, dunque il programmatore dovrà assicurarsi di aggiungerlo manualmente in modo da garantire che `s1` rimanga una stringa valida;
- nel caso in cui **la stringa di sorgente contenga un numero di caratteri minore di `n`**, i caratteri restanti all'interno della stringa di destinazione `s1` verranno occupati da caratteri di fine stringa.

La funzione **`strncat`** viene utilizzata per **concatenare un certo numero di caratteri di una stringa con un'altra**. Una chiamata alla funzione `strncat` rispetta la seguente sintassi:

```
strncat(s1, s2, n)
```

dove `s1` è la stringa di destinazione, dunque la stringa a cui verranno accodati i caratteri, `s2` è la stringa di sorgente, da cui provengono i caratteri che verranno concatenati al termine di `s1`, mentre **`n`** è il numero di caratteri da concatenare a `s1`. Il **valore ritornato** dalla funzione è un **puntatore alla stringa di destinazione**. Come per `strcat`, è importante **assicurarsi che la stringa di destinazione possa contenere abbastanza caratteri da ospitare completamente la concatenazione**; a differenza di `strncpy`, però, la funzione cercherà sempre di aggiungere un carattere di fine stringa dopo aver effettuato la concatenazione (a patto che abbia spazio per farlo).

La funzione **`strncmp`** viene utilizzata per **confrontare un certo numero di caratteri di due stringhe in base al loro ordine lessicografico**. Una chiamata alla funzione `strncmp` rispetta la seguente sintassi:

```
strncmp(s1, s2, n)
```

dove `s1` e `s2` sono le due stringhe da confrontare, mentre **`n`** è il numero di caratteri da confrontare. Il confronto, e di conseguenza il valore ritornato, sono perfettamente analoghi a quelli della funzione `strcmp`. Se almeno una delle due stringhe confrontate ha una lunghezza minore di `n`, allora il confronto terminerà non appena si incontra il primo carattere di fine stringa.
___
##### Input e output di stringhe

Per l'**input** e **output di stringhe di testo**, oltre ai classici `printf` e `scanf`, si possono utilizzare altre funzioni più versatili, tra cui:
- **`putchar`**;
- **`puts`**;
- **`fputs`**;
- **`getchar`**;
- **`gets`**;
- **`fgets`**.

Si tratta di funzioni tutte contenute nell'header file **`stdio.h`**; in particolare, possiamo suddividere l'elenco appena fornito tra le funzioni di output (le prime 3, identificabili dalla presenza della parola `puts` nel loro identificatore) e le funzioni di input (le ultime 3, identificabili dalla presenza della parola `gets` nel loro identificatore).

Partiamo dalle funzioni di **output di stringhe e caratteri**, e in particolare dalla funzione **`putchar`**. Quest'ultima viene utilizzata per **stampare un singolo carattere a schermo**. La sinossi completa della funzione è:

```
int putchar(int c)
```

Il parametro **`c`**, di tipo `int`, consiste nel **codice ASCII** del carattere che si vuole stampare (la funzione prenderà i bit meno significativi dell'intero inserito). Ci si potrebbe, però, chiedere **perché il tipo di ritorno è `int`?** Normalmente, la funzione restituisce proprio `c`, ossia l'intero che prende in input e che rappresenta il codice ASCII del carattere stampato, ma in alcuni casi particolari potrebbe restituire **`EOF`**, una "macro" definita in `stdio.h` che rappresenta un errore o la fine di un flusso di dati, e che tipicamente ha valore $-1$; il valore speciale `EOF` viene restituito, dunque, quando l'operazione di stampa fallisce, eventualità che si verifica quando:
- per qualche motivo, il programma in esecuzione ha **chiuso il flusso `stdout`**;
- per **problemi fisici o del sistema operativo** (ad esempio, se l'output viene reindirizzato su un file ma il disco è pieno, oppure se ci sono problemi di [[SO2_02 - File system#Permessi di accesso ai file|permessi]] di scrittura);
- **si sta scrivendo su un terminale remoto** e **la connessione cade improvvisamente**.

Di seguito, alcuni **esempi di utilizzo** della funzione `putchar`:

```
putchar('i');                 // stampa il carattere 'i'


putchar('\n');                // va a capo


char lettera = 'H';
putchar(lettera);             // stampa il carattere 'H'


for (char c = 'a'; c <= 'z'; c++)
{
	putchar(c);               // stampa, una per una, le lettere dell'alfabeto
}


int esito = putchar('A');
if (esito == EOF)             // se ci sono stati errori nella stampa...
{
	printf("ERRORE CRITICO: impossibile scrivere su stdout");
	return 1;
}
```

Passiamo, ora, alla funzione **`puts`**, che viene utilizzata per **stampare una stringa a schermo**. La sinossi completa della funzione è:

```
int puts(const char *s)
```

Il parametro **`s`** rappresenta il puntatore alla stringa da stampare (sia essa uno "string literal" o una variabile contenente un array di caratteri). Sostanzialmente, quello che fa la funzione è **stampare la stringa carattere per carattere** (in modo equivalente, insomma, a chiamare ciclicamente `putchar`) **finché non incontra il carattere di fine stringa `\0`**; perciò, è fondamentale che la stringa `s` contenga effettivamente tale carattere. Per quanto riguarda il **valore di ritorno**, se la funzione viene eseguita senza problemi esso è un **numero intero non negativo** (spesso il numero di caratteri stampati), mentre se l'esecuzione fallisce per qualche motivo viene restituito un valore **`EOF`**, come avverrebbe per `putchar`.

Per adesso, la funzione `puts` potrebbe sembrare solo **un'altra versione di [[SO2_04 - C#Output|printf]]**, e se vogliamo dirla tutta anche meno potente dato che **`puts` può solo stampare messaggi semplici, dunque non formattabili**. Allora che utilità può avere questa funzione? Innanzitutto, un pregio piccolo ma che può tornare utile è che **aggiunge in automatico un carattere di "a capo"** (**`\n`**) al termine della stringa che viene stampata; passando a cose più importanti, uno dei principali punti a favore di `puts` è la **maggiore efficienza rispetto a `printf`**, dato che quest'ultima, proprio per la sua natura dinamica, risulta anche essere relativamente pesante a livello computazionale; inoltre, utilizzare `puts` piuttosto che `printf` permette di **prevenire la "Format String Vulnerability"** (in parole povere, con `printf` un utente malintenzionato potrebbe inserire stringhe formattate costruite in modo tale da leggere zone di memoria altrimenti inaccessibili). Insomma, in generale:
- utilizzare `printf` quando si devono combinare stringhe di testo con variabili;
- utilizzare `puts` negli altri casi, in cui si vuole stampare un messaggio "fisso" o una variabile contenente una stringa.

Vediamo, a questo punto, l'ultima delle funzioni di output che analizzeremo, ossia **`fputs`**, che viene utilizzata per **stampare una stringa su qualsiasi stream di dati** supportato da C. La sinossi completa della funzione è:

```
int fputs(const char *s, FILE *stream)
```

dove **`s`** rappresenta, come in `puts`, un puntatore alla stringa da stampare o scrivere, mentre **`stream`** consiste nel flusso di dati verso cui si vuole inviare la stringa `s`. Come anticipato, tale stream può essere uno qualsiasi di quelli supportati, ad esempio:
- un **[[SO2_04 - C#File|file]]** precedentemente aperto;
- **`stdout`**, o "**standard output**", ossia sostanzialmente lo schermo (in tal caso, il risultato sarebbe pressoché identico a una stampa tramite `printf` o `puts`);
- **`stderr`**, o "**standard error**", ossia il flusso utilizzato per stampare a schermo messaggi di errore.

Analogamente a `puts`, `fputs` restituisce un valore intero non negativo se la stampa è andata a buon fine, e il valore `EOF` se essa fallisce.

A livello di funzionamento effettivo, la **differenza principale tra `puts` e `fputs`**, oltre alla maggior flessibilità della seconda, è che quest'ultima **non aggiunge automaticamente un "a capo"** al termine della stringa stampata. Ciò è previsto in vista della scrittura in un file, dato che naturalmente si può voler comporre una riga dello stesso "a pezzi", mediante più chiamate a `fputs`, cosa che sarebbe impossibile se si andasse a capo a ogni chiamata.

Di seguito, alcuni **esempi di utilizzo** della funzione `fputs`:

```
fputs("Prima riga - ", stdout);
fputs("Seconda riga\n", stdout);       // le due stampe avverranno sulla stessa riga


FILE *mio_file = fopen("prova.txt", "w");

if (mio_file == NULL)
{
	fputs("ERRORE CRITICO: impossibile creare il file\n", stderr);
	return 1;
}

fputs("Hello World!\n", mio_file);
fclose(mio_file);
```

È il momento di passare alle funzioni di **input di stringhe e caratteri**, cominciando con la funzione **`getchar`**, il cui scopo è quello di **leggere un singolo carattere dall'input dell'utente**, e dunque dallo "**standard input**", o **`stdin`**. La sinossi completa della funzione è:

```
int getchar()
```

Dunque, `getchar` **non accetta argomenti** in input, e il suo **valore di ritorno** è invece il **codice ASCII del carattere letto** in caso di successo, o ancora il valore speciale `EOF` in caso di fallimento (ad esempio quando l'input da `stdin` termina improvvisamente). 

Una particolarità del funzionamento di `getchar` è il cosiddetto "**buffering**": sostanzialmente, la funzione non rileva un carattere non appena esso viene digitato sulla tastiera; piuttosto, non appena viene chiamata `getchar`, il programma si mette "in attesa" e **ogni volta che un carattere viene digitato, viene inserito in un buffer, finché non si preme invio** (**`\n`**); solo a quel punto, infatti, il programma ritorna "in esecuzione", e `getchar` estrarrà dal buffer il primo carattere inserito al suo interno, restituendolo. Dunque, altri eventuali caratteri digitati prima di `\n` rimarranno nel buffer, e verranno eventualmente estratti in seguito con altre chiamate a `getchar` o a `scanf`.

Di seguito, alcuni **esempi di utilizzo** della funzione `getchar`:

```
int c; 

printf("Digita un carattere: ");
c = getchar();                    // Legge il primo carattere digitato


int eta; 
char iniziale; 

printf("Quanti anni hai? "); 
scanf("%d", &eta);          

// Ora nel buffer c'è il numero, ma anche l'invio '\n'. Dunque, occorrerà
// "svuotare il buffer", leggendo e scartando caratteri finché non si trova '\n' o 
// la fine dell'input 

int c; 
while ((c = getchar()) != '\n' && c != EOF) 
{ 
	
}

printf("Qual e' l'iniziale del tuo nome? "); 
iniziale = getchar();              // Ora funzionerà accuratamente


int c; 
printf("Scrivi una frase (premi Invio per terminare):\n");

while ((c = getchar()) != '\n' && c != EOF) 
{
	if (c >= 'a' && c <= 'z') 
	{ 
		putchar(c - 32);
	} 
	else
	{ 
		putchar(c);
	} 
}
```

Ora, per quanto riguarda la funzione **`gets`**, non andremo eccessivamente nel dettaglio perché si tratta di una **funzione "deprecata"**, che si consiglia di non utilizzare mai (è stata deprecata dal C99 e completamente rimossa dallo standard C11, tanto che la maggior parte dei compilatori C lanceranno un avvertimento se si cerca di compilare tale istruzione). La sinossi completa della funzione è:

```
char *gets(char *s)
```

L'idea alla base del funzionamento di `gets` è molto semplice: teoricamente, essa permette di **leggere caratteri dall'`stdin` finché l'utente non preme invio**, e di salvare tutti i caratteri letti (escluso lo `\n` finale) nell'array di caratteri a cui punta `s`, aggiungendo automaticamente il carattere di fine stringa `\0` al termine della stessa; in seguito, restituirebbe il puntatore alla stringa ottenuta in caso di successo, o un valore nullo `NULL` in caso contrario. Il problema con questa funzione deriva da un fatto molto semplice: **`gets` riceve solo il puntatore di inizio del blocco di memoria dove scrivere, ma non quanto sia grande**. Dunque, se le dimensioni dell'input eccedono quelle della memoria disponibile, avviene quello che viene chiamato "**buffer overflow**", scenario che può portare a crash del programma, alla sovrascrittura di dati importanti o anche a vulnerabilità di sicurezza.

Infine, parliamo della funzione **`fgets`**, la cui sinossi completa è:

```
char *fgets(char *s, int size, FILE *stream)
```

Possiamo vederla come una **versione migliore e più sicura di `gets`**, che quindi si occupa di **leggere una stringa di testo (inclusi gli spazi) da `stdin`**. Una chiamata a tale funzione deve contenere i seguenti argomenti:
- **`s`**, ossia un puntatore all'array di caratteri dove salvare il testo letto in input;
- **`size`**, ossia la dimensione totale dell'array di caratteri allocato (in altre parole, il numero massimo di byte a disposizione);
- **`stream`**, ossia il flusso di dati da cui leggere il testo (se si vuole leggere dalla tastiera, si inserirà `stdin`; altrimenti, se si vuole leggere da un file, si inserirà un puntatore a tale file).

La funzione `fgets` smette di leggere caratteri appena si verifica una di queste tre condizioni:
- viene raggiunto il **limite di sicurezza**, ossia vengono letti esattamente `size - 1` caratteri (si conserva, infatti, l'ultima posizione per il carattere di fine stringa `\0`);
- si incontra un "**a capo**" (**`\n`**), ossia l'utente preme invio oppure si termina una riga del file;
- si incontra la **fine del file** (**`EOF`**), e dunque non ci sono più dati da leggere.

C'è, però, da fare una precisazione importante riguardo `fgets`: **se essa smette di leggere perché l'utente ha premuto invio, verrà letto e memorizzato anche il carattere `\n`**. Questo può essere un problema nel momento in cui si vogliono stampare stringhe formattate in modi precisi o si vogliono salvare esclusivamente i dati effettivi. La soluzione più "pulita" ed elegante a questo problema è **rimuovere manualmente quel carattere `\n`**, utilizzando una funzione chiamata **`strcspn`**, inclusa in `string.h`: tramite questa funzione, è possibile cercare la prima occorrenza di un carattere all'interno di una stringa, e restituire la sua posizione, e nel nostro caso il carattere che vogliamo cercare è `\n`. Dunque:

```
char nome[20]; 

printf("Come ti chiami? "); 
fgets(nome, 20, stdin);

nome[strcspn(nome, "\n")] = '\0';
```

## Puntatori e allocazione dinamica di memoria

##### Puntatori

Per poter parlare di **allocazione dinamica di memoria**, è necessario approfondire un concetto che abbiamo già utilizzato spesso finora: il "**puntatore**". Concretamente, un puntatore è una **variabile che contiene l'indirizzo in memoria di un'altra variabile**; per **dichiarare un puntatore** si utilizza la seguente sintassi:

```
type *name;
```

dove **`type`** è il **tipo del dato a cui punta il puntatore** che si sta dichiarando, mentre **`name`** sarà l'**identificatore del puntatore** in questione. Si noti che la sintassi è perfettamente identica a quella della [[SO2_04 - C#Dichiarazione e tipi di dati|dichiarazione di una normale variabile]], tranne per l'aggiunta di un **asterisco** (**`*`**) prima dell'identificatore. Ad esempio, di seguito si dichiarano due puntatori, il primo a un carattere e il secondo a un intero:

```
char *dest;
int *intPtr;
```

Per lavorare con i puntatori, è fondamentale conoscere il funzionamento degli **operatori `*` e `&`**:
- l'operatore **`*`**, detto anche "**operatore di deferenziazione**", permette di **accedere al dato a cui punta un puntatore**, dunque di accedere non all'indirizzo di memoria effettivamente contenuto nel puntatore ma al dato contenuto in tale indirizzo della memoria principale;
- l'operatore **`&`**, detto anche "**operatore address-of**", permette di **ottenere l'indirizzo di memoria di una variabile**, dunque l'indirizzo in cui è contenuto il dato contenuto in tale variabile.

Ad esempio, potremmo scrivere:

```
int num = 23;

int *ptr;
ptr = &num;
```

per inizializzare il puntatore `ptr` con l'indirizzo di memoria di `num`. Al tempo stesso, potremmo scrivere:

```
*ptr = 99;
```

per modificare il dato che si trova all'indirizzo di memoria contenuto in `ptr` (ossia, il dato contenuto nella variabile `num`), assegnandoci il numero 99.

Un puntatore, in generale, può essere visto come una coppia di un valore diretto e un valore indiretto: con "**valore diretto**" si intende il **valore effettivamente contenuto nel puntatore**, ossia un certo indirizzo di memoria (si accede al valore diretto di un puntatore semplicemente riferendosi al suo identificatore); con "**valore indiretto**", invece, si intende il **dato presente all'indirizzo di memoria contenuto nel puntatore** (si accede al valore indiretto di un puntatore tramite una deferenziazione).

Naturalmente, essendo un puntatore nient'altro che un numero intero, è possibile effettuare alcune **operazioni aritmetiche** su di esso; si tenga, tuttavia, a mente che esse funzioneranno in modo leggermente diverso di come ci si potrebbe aspettare. In poche parole, potremmo dire che le operazioni aritmetiche sono svolte "in scala", **in base alla dimensione del tipo di dato a cui punta il puntatore**. Ad esempio, supponendo di avere un puntatore a un `int`, che occupa tendenzialmente 4 byte in memoria, incrementare di 1 tale puntatore equivarrà a incrementare l'indirizzo contenuto al suo interno di 4, ossia di `1 * sizeof(int)`. In generale, supponendo di incrementare lo stesso puntatore di un numero $n$ generico, il puntatore verrà effettivamente incrementato di $4n$.

In C, **la variabile contenente un [[SO2_04 - C#Array|array]] e un puntatore al primo elemento di tale array sono esattamente la stessa cosa**: ciò vuol dire che le seguenti due istruzioni:

```
ptr = &arr[0];
ptr = arr;
```

sono equivalenti. Del resto, la sintassi tipica degli array con le parentesi quadre, ad esempio `arr[i]`, è solo un modo più semplice di scrivere `*(arr + i)`.

##### Allocazione dinamica di memoria

Come abbiamo detto parlando dei [[SO2_03 - Processi#Come sono rappresentati i processi?|processi]], quando un programma viene eseguito l'OS gli riserva una porzione di memoria principale, divisa in sei aree. Tra queste 6 aree, quella che ci interessa in questo momento è l'**heap**, quello destinato all'**allocazione dinamica di memoria**. Ma cosa si intende quando ci si riferisce a questo concetto?

Allocare dinamicamente della memoria vuol dire **allocare memoria durante l'esecuzione del programma**, spesso in base a decisioni prese a runtime dall'utente esecutore. Per allocare memoria dinamicamente, utilizzeremo principalmente le seguenti tre [[SO2_04 - C#Funzioni|funzioni]], contenute nell'header file **`stdlib.h`**:
- **`void *malloc(size_t size)`**;
- **`void *calloc(size_t nmemb, size_t size)`**;
- **`void free(void *ptr)`**.

Vediamo nel dettaglio come funziona ciascuna di queste funzioni. La funzione **`malloc`** (abbreviazione di "**Memory Allocation**") chiede all'OS di **riservare un blocco di memoria di `size` byte**, e se tale operazione ha successo restituisce un **puntatore all'inizio del blocco di memoria riservato**; in caso di fallimento, invece, restituisce un **valore nullo `NULL`**. Una particolarità da ricordare riguardo il funzionamento di `malloc`: nonostante serva per riservare al processo della memoria, **`malloc` non pulisce la memoria assegnata**, ed essa sarà dunque costituita da "dati spazzatura", a meno che non si agisca su di essi.

La funzione **`calloc`** (abbreviazione di "**Contiguous Allocation**") chiede all'OS di **riservare un blocco di memoria contigua in vista di [[SO2_04 - C#Array|array]] dinamici**: per fare ciò, infatti, si richiede il numero di elementi **`nmemb`** e la dimensione **`size`** di ciascuno di tali elementi. Per quanto riguarda i valori di ritorno, valgono le stesse identiche considerazioni fatte per `malloc`. Una differenza cruciale tra `malloc` e `calloc` è che quest'ultima **pulisce la memoria allocata, inizializzandola tutta a `0`**.

Infine, abbiamo la funzione **`free`**. Mentre `malloc` e `calloc` vengono utilizzate per allocare memoria nell'heap, `free` viene utilizzato per **liberare memoria e "restituirla" all'OS**. Infatti, a differenza di linguaggi come Python e Java, **C non dispone di un "garbage collector"**, ossia di un modo automatizzato di liberare la memoria inutilizzata, e dunque tale responsabilità ricade sul programmatore. La funzione `free` serve proprio a questo, e prende come unico argomento un **puntatore `ptr`** all'area di memoria da liberare, che **deve coincidere con il puntatore originariamente restituito da `malloc` o da `calloc`**. Alcune cose utili da ricordare contestualmente alla funzione `free`:
- se si inserisce come argomento di `free` un puntatore non restituito da `calloc` o `malloc`, il comportamento di `free` è indefinito;
- se si chiama `free` più di una volta su uno stesso puntatore si otterranno risultati indefiniti.

Per evitare, in particolare, l'ultima evenienza, è opportuno assegnare un valore nullo `NULL` a un puntatore dopo averlo passato come input a una chiamata `free`, dato che quest'ultima non eseguirà alcuna operazione se il puntatore preso in input è `NULL`.

## Strutture

In C, esiste un modo per raggruppare più variabili di tipi diversi: la "**struttura**". Una struttura può essere vista semplicemente come un **contenitore eterogeneo di variabili**, tutte **raggruppate sotto un unico nome**. Questo costrutto può essere utile in vari contesti in cui, più che vari dati semplici, si vogliono modellare delle "entità", a cui associare più dati di tipi diversi.

##### Definizione di una struttura

Prima di utilizzare una struttura, naturalmente, è necessario **definirla**. Ciò può avvenire principalmente in 3 modi, in base alle situazione e alle esigenze:
- tramite una **variabile struttura**;
- tramite una **struttura etichettata**;
- tramite una **struttura type-defined**.

Definire una **variabile struttura** vuol dire definire una sorta di "**struttura anonima**", ossia non una struttura generica e che potrà essere "istanziata" più volte, ma piuttosto una **struttura che viene definita "localmente" a una variabile e subito assegnata alla stessa**. La definizione di una variabile struttura segue la seguente forma generica:

```
struct {

	// ...
	
} variable_name;
```

dove **`variable_name`** è l'identificatore della variabile a cui si sta assegnando la struttura appena definita. Tra le parentesi graffe, invece, saranno elencate le variabili che apparterranno a tale struttura (dette "**membri**" della struttura), e per ciascuna di esse si dovrà specificare tipo e identificatore. Ad esempio, scrivere:

```
struct {
	double x;
	double y;
} point2D;
```

definisce una struttura i cui membri sono due numeri reali di tipo `double`, nominati `x` e `y`, e assegna tale struttura alla variabile `point2D`. Si tratta di un metodo comodo e veloce se si sa a priori che **tale struttura servirà una sola volta** in tutto il programma, ma altrimenti può essere poco pratica.

Definire una **struttura etichettata** è simile al dichiarare una variabile struttura, ma la differenza è che una struttura etichettata rappresenta effettivamente una **generalizzazione di un'entità**, e dunque potrà essere riutilizzata e "istanziata" per quante variabili si desidera (per fare ciò, basterà dichiarare variabili aventi come tipo **`struct Struct_Name`**, dove `Struct_Name` è l'identificatore assegnato alla struttura etichettata al momento della sua dichiarazione). La definizione di una struttura etichettata segue la seguente forma generica:

```
struct Struct_Name {

	// ...

};
```

Ad esempio, scrivere:

```
struct Punto2D {
	double x;
	double y;
};

struct Punto2D p1;
struct Punto2D p2;
```

definisce una struttura `Punto2D` avente come membri due numeri reali di tipo `double`, nominati `x` e `y`, e dichiara due variabili `p1` e `p2` entrambe di tipo `Punto2D`, dunque costituite in realtà da una coppia particolare di valori `x` e `y`. Naturalmente, utilizzare una struttura etichettata ha come principale pregio la sua **riutilizzabilità**.

Definire una **struttura type-defined**, infine, è l'approccio più comune e moderno, e consiste nel **definire una struttura generica come se fosse un nuovo [[SO2_04 - C#Dichiarazione e tipi di dati|tipo di dato]]**. La definizione di una struttura type-defined segue la seguente forma generica:

```
typedef struct {
	
	// ...
	
} Type_Name
```

dove **`Type_Name`** è l'identificatore assegnato al nuovo "tipo" dichiarato. Ad esempio, scrivere:

```
typedef struct {
	double x;
	double y;
} Punto2D;

Punto2D p1;
Punto2D p2;
```

definisce una struttura `Punto2D` come un nuovo tipo di dato, che ha come membri due numeri reali di tipo `double`, nominati `x` e `y`, e in seguito dichiara due variabili proprio di tipo `Punto2D`. Questo approccio, molto simile ma sottilmente diverso da quello delle strutture etichettate, permette di avere una sintassi ancora più pulita (non si dovrà più aggiungere la keyword `struct` a ogni dichiarazione di variabile), oltre alla già citata riutilizzabilità.
___
##### Inizializzazione di una struttura e accesso ai membri

Finora, abbiamo visto solo come [[SO2_04 - C#Definizione di una struttura|definire una struttura]], ma mai come effettuare l'**inizializzazione di una struttura**, ossia come **assegnare dei valori concreti ai membri di una struttura**. Per fare ciò, si possono scegliere due mezzi:
- l'**inizializzazione posizionale**;
- i **designatori** (disponibili da C99 in poi).

Inizializzare una struttura tramite l'**inizializzazione posizionale** vuol dire semplicemente **specificare i valori di ogni membro in una sequenza ordinata**, che deve dunque seguire la stessa sequenza in cui sono stati inseriti i membri al momento della definizione. Ad esempio, tornando alla struttura `Punto2D` che abbiamo considerato finora, scrivere:

```
Punto2D p1 = {10, 20};
```

assegna il valore 10 al membro `x` e il valore 20 al membro `y`, poiché `x` è il primo membro specificato nella definizione e dunque verrà associato al primo "argomento", e analogamente `y` è il secondo membro e dunque verrà associato al secondo "argomento". In alternativa, è possibile utilizzare i **designatori**: in questo modo, si dovrà **specificare a quale membro assegnare ogni argomento in modo esplicito**, ma in compenso fare ciò permette di **trascendere qualsiasi ordine** e di inserire gli argomenti come si desidera. Ad esempio:

```
Punto2D p2 = { .y = 50, .x = 5 };
```

Indipendentemente dall'approccio scelto, **è possibile non inizializzare tutti i membri**: nel caso dell'inizializzazione posizionale, indicare meno argomenti di quanti sono i membri vorrà dire che saranno gli ultimi membri della struttura a non disporre di un valore; invece, nel caso dei designatori, i membri privi di valore saranno quelli che non sono stati specificati durante l'inizializzazione.

A questo punto, una volta inizializzati i membri, sarà possibile **accedere a tali membri**: per fare ciò, basterà utilizzare un **punto fermo** (**`.`**), che andrà a seguire l'identificatore della variabile che ospita la struttura e a precedere il nome del membro a cui si vuole accedere. Ad esempio:

```
p1.x = 22;
p2.y = 9.4;
```
___
##### Allocazione dinamica di strutture

Quando le strutture in un programma diventano di grandi dimensioni, o troppo numerose, allocarle nello Stack può diventare rischioso e inefficiente. In questi contesti, è possibile utilizzare l'**[[SO2_04 - C#Allocazione dinamica di memoria|allocazione dinamica di memoria]] per creare strutture**.

Tipicamente, l'allocazione dinamica contestualmente alle strutture segue i seguenti passaggi:
- viene **[[SO2_04 - C#Definizione di una struttura|definita la struttura]]**;
- viene **dichiarato un [[SO2_04 - C#Puntatori|puntatore]]** a quel tipo di struttura;
- si utilizza `malloc` o `calloc` per **allocare l'esatta memoria necessaria per quella struttura**, utilizzando `sizeof`.

Ad esempio:

```
typedef struct {
	char nome[50];
	int eta;
} utente;

int main() 
{
	Utente *ptrUtente;
	ptrUtente = malloc(sizeof(Utente));

	return 0;
}
```

A questo punto, però, abbiamo solo allocato memoria per una struttura ma non abbiamo effettivamente dei valori per i vari membri della stessa. Nel fare ciò, specifichiamo che **quando una struttura viene allocata dinamicamente, non si utilizza più il punto `.` ma una freccia `->` per accedere ai membri**. Alternativamente, se si preferisce utilizzare comunque il punto, si dovrà prima **effettuare una dereferenziazione e poi accedere al membro**. Dunque, le seguenti due istruzioni sono perfettamente equivalenti ed entrambe valide:

```
ptrUtente->nome
(*ptrUtente).nome
```
___
##### Passaggio di strutture come parametri di una funzione

In alcuni contesti, può servire **passare una struttura come parametro di una [[SO2_04 - C#Funzioni|funzione]]**. Per fare ciò, C permette due approcci distinti:
- passare la struttura per **valore**;
- passare la struttura per **riferimento** (per intenderci, si intende tramite [[SO2_04 - C#Puntatori|puntatore]]).

Passare una struttura **per valore** fa in modo che **la funzione riceva una copia completa della struttura**. Questo approccio è sicuramente più semplice e intuitivo, e dato che la funzione lavora su una copia non c'è il rischio che i dati della struttura originale vengano modificati per errore; al tempo stesso, però, si tratta di un approccio abbastanza inefficiente, sia a livello di memoria (lo Stack dovrà ospitare ancora un'altra struttura) che a livello di tempo. Tornando sulla struttura di esempio vista nel [[SO2_04 - C#Allocazione dinamica di strutture|paragrafo precedente]], passare la struttura `Utente` per valore a un'ipotetica funzione `stampaUtente` ha la seguente forma:

```
void stampaUtente(Utente u) 
{
		printf("%s ha %d anni\n", u.nome, u.eta);
}
```

Passare una struttura **per riferimento** fa in modo che **la funzione riceva solamente il puntatore alla struttura**, e dunque vada a lavorare direttamente su di essa. Questo approccio rappresenta una grande ottimizzazione, dato che permette di non riempire ancor di più lo Stack ed è molto più efficiente; al tempo stesso, però, c'è il rischio che i dati originali vengano modificati (se si vuole prevenire ciò, si dovrà inserire la parola chiave `const`). Ad esempio:

```
void stampaUtente(Utente *u)
{
	printf("%s ha %d anni\n", u->nome, u->eta);
}
```

oppure:

```
void stampaUtente(const Utente *u)
{
	printf("%s ha %d anni\n", u->nome, u->eta);
}
```
